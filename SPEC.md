# SPEC.md

## Problem Statement and Vision

AI agents are producing increasing volumes of code, but there is no standardized way to prove authorship, verify integrity, or trace the provenance of agent-generated software. This creates trust, security, and auditing problems as AI-authored code enters production systems.

**agentmark** is a cryptographic provenance system that enables AI agents to sign their code, verify signatures, and maintain immutable audit trails. It provides cryptographic proof that specific code was generated by a specific AI agent at a specific time, creating a foundation for trust in AI-authored software.

## Agent User Stories

**Primary Users: AI Agents**
- As a code-generating agent, I need to sign my output so users can verify I authored it
- As a code-reviewing agent, I need to verify which agent authored code before analyzing it
- As a deployment agent, I need to check code provenance before deploying to production

**Secondary Users: Human Developers**
- As a developer, I need to audit which AI agents contributed to my codebase
- As a security engineer, I need to trace suspicious code back to its generating agent
- As a compliance officer, I need immutable logs of code authorship for regulatory purposes

## Public API Contract

### Core Functions

```python
def sign_content(
    content: str,
    agent_id: str,
    private_key_pem: str,
    timestamp: Optional[float] = None
) -> Dict[str, Any]:
    """
    Sign content with cryptographic signature.
    
    Args:
        content: The code or text to sign
        agent_id: Unique identifier for the signing agent
        private_key_pem: RSA private key in PEM format
        timestamp: Unix timestamp (defaults to time.time())
    
    Returns:
        {
            "content_hash": str,  # SHA-256 hex digest
            "agent_id": str,
            "timestamp": float,
            "signature": str,     # Base64-encoded RSA signature
            "algorithm": "RSA-SHA256"
        }
    
    Raises:
        ValueError: Invalid private key or agent_id format
        TypeError: content is not string
    """

def verify_signature(
    content: str,
    signature_data: Dict[str, Any],
    public_key_pem: str
) -> Dict[str, Any]:
    """
    Verify cryptographic signature of content.
    
    Args:
        content: Original content to verify
        signature_data: Output from sign_content()
        public_key_pem: RSA public key in PEM format
    
    Returns:
        {
            "valid": bool,
            "agent_id": str,
            "timestamp": float,
            "content_hash_matches": bool,
            "signature_valid": bool,
            "error": Optional[str]
        }
    
    Raises:
        ValueError: Invalid public key format
        TypeError: Malformed signature_data
    """

def generate_keypair(agent_id: str) -> Tuple[str, str]:
    """
    Generate RSA-2048 key pair for agent.
    
    Args:
        agent_id: Unique agent identifier
    
    Returns:
        (private_key_pem, public_key_pem)
    
    Raises:
        ValueError: Invalid agent_id format (must be [a-zA-Z0-9_-]+)
    """

def log_provenance_entry(
    signature_data: Dict[str, Any],
    file_path: str,
    log_file_path: str = "provenance.log"
) -> None:
    """
    Append provenance entry to log file.
    
    Args:
        signature_data: Output from sign_content()
        file_path: Path to signed file
        log_file_path: Path to provenance log file
    
    Raises:
        IOError: Cannot write to log file
        ValueError: Malformed signature_data
    """

def verify_log_integrity(log_file_path: str) -> Dict[str, Any]:
    """
    Verify integrity of provenance log chain.
    
    Args:
        log_file_path: Path to provenance log file
    
    Returns:
        {
            "valid": bool,
            "entry_count": int,
            "broken_chain_at": Optional[int],
            "error": Optional[str]
        }
    """
```

### CLI Interface

```bash
# Generate key pair
agentmark keygen --agent-id <agent_id> [--output-dir <dir>]

# Sign content
agentmark sign <file_path> --key <private_key_path> --agent-id <agent_id> [--output <signature_file>]

# Verify signature
agentmark verify <signature_file> --content <file_path> --public-key <public_key_path>

# Log to provenance chain
agentmark log <signature_file> --file-path <path> [--log-file <log_path>]

# Verify log integrity
agentmark verify-log [--log-file <log_path>]

# Show signature info
agentmark info <signature_file>
```

## Data Schemas

### Signature Data Schema

```json
{
  "content_hash": "string (64-char hex SHA-256)",
  "agent_id": "string (regex: [a-zA-Z0-9_-]{1,64})",
  "timestamp": "number (Unix timestamp with millisecond precision)",
  "signature": "string (Base64-encoded bytes)",
  "algorithm": "string (fixed: 'RSA-SHA256')",
  "version": "string (semver, e.g., '1.0.0')"
}
```

### Provenance Log Entry Schema

```json
{
  "timestamp": "number (Unix timestamp)",
  "agent_id": "string",
  "file_path": "string (relative or absolute path)",
  "content_hash": "string (64-char hex SHA-256)",
  "signature": "string (Base64-encoded)",
  "parent_hash": "string (SHA-256 of previous log entry, null for first entry)",
  "entry_hash": "string (SHA-256 of this entry excluding entry_hash field)"
}
```

### Key File Formats

```
# Private Key File: <agent_id>.private.pem
-----BEGIN RSA PRIVATE KEY-----
[Base64 encoded key data]
-----END RSA PRIVATE KEY-----

# Public Key File: <agent_id>.public.pem  
-----BEGIN PUBLIC KEY-----
[Base64 encoded key data]
-----END PUBLIC KEY-----
```

## Error Handling Contract

### Error Categories

1. **CryptographicError**: Key generation, signing, or verification failures
2. **ValidationError**: Schema validation, format, or data integrity issues  
3. **IOError**: File system access problems
4. **ConfigurationError**: Invalid agent IDs, paths, or configuration

### Error Response Format

All functions return errors in consistent format:

```python
{
    "error_type": "ValidationError",
    "error_code": "INVALID_AGENT_ID", 
    "message": "Agent ID must match [a-zA-Z0-9_-]{1,64}",
    "details": {"provided": "invalid@agent", "field": "agent_id"}
}
```

### CLI Exit Codes

- `0`: Success
- `1`: Validation error (bad input)
- `2`: Cryptographic error (key/signature problem)
- `3`: IO error (file access problem)
- `4`: Configuration error

## Extension Points

Future agents can extend agentmark capabilities through these explicit interfaces:

### 1. Signature Algorithm Adapters

```python
# Location: agentmark/algorithms/
class SignatureAlgorithm(ABC):
    @abstractmethod
    def get_algorithm_name(self) -> str:
        """Return algorithm identifier (e.g., 'ED25519-SHA256')"""
        
    @abstractmethod  
    def generate_keypair(self) -> Tuple[str, str]:
        """Return (private_key, public_key) as PEM strings"""
        
    @abstractmethod
    def sign(self, content_hash: bytes, private_key: str) -> bytes:
        """Sign hash with private key, return signature bytes"""
        
    @abstractmethod
    def verify(self, content_hash: bytes, signature: bytes, public_key: str) -> bool:
        """Verify signature against hash and public key"""

# Registration:
agentmark.register_algorithm('ed25519', Ed25519Algorithm())
```

### 2. Storage Backend Adapters

```python  
# Location: agentmark/storage/
class StorageBackend(ABC):
    @abstractmethod
    def write_log_entry(self, entry: Dict[str, Any]) -> None:
        """Persist log entry to backend"""
        
    @abstractmethod
    def read_log_entries(self, limit: Optional[int] = None) -> List[Dict[str, Any]]:
        """Read entries from backend"""
        
    @abstractmethod
    def verify_integrity(self) -> Dict[str, Any]:
        """Check backend integrity, return status"""

# Registration:
agentmark.register_storage('s3', S3StorageBackend(bucket='provenance'))
```

### 3. Key Management Providers

```python
# Location: agentmark/keymanagers/
class KeyManager(ABC):
    @abstractmethod
    def store_private_key(self, agent_id: str, private_key: str) -> str:
        """Store private key, return key identifier"""
        
    @abstractmethod
    def retrieve_private_key(self, key_id: str) -> str:
        """Retrieve private key by identifier"""
        
    @abstractmethod
    def get_public_key(self, agent_id: str) -> str:
        """Get public key for agent"""

# Registration:
agentmark.register_key_manager('vault', HashiCorpVaultKeyManager())
```

### 4. Log Format Handlers

```python
# Location: agentmark/formats/
class LogFormat(ABC):
    @abstractmethod
    def serialize_entry(self, entry: Dict[str, Any]) -> bytes:
        """Convert entry to storage format"""
        
    @abstractmethod  
    def deserialize_entry(self, data: bytes) -> Dict[str, Any]:
        """Parse entry from storage format"""
        
    @abstractmethod
    def get_format_name(self) -> str:
        """Return format identifier"""

# Registration:
agentmark.register_format('protobuf', ProtobufLogFormat())
```

### 5. CLI Command Extensions

```python
# Location: agentmark/commands/
class Command(ABC):
    @abstractmethod
    def get_command_name(self) -> str:
        """Return CLI command name"""
        
    @abstractmethod

        """Add command-specific arguments"""

```

## Security Constraints

### Cryptographic Requirements

1. **RSA key size**: Minimum 2048 bits
2. **Hash algorithm**: SHA-256 or stronger
3. **Random number generation**: Use `os.urandom()` for all crypto operations
4. **Key storage**: Private keys stored in PEM format with appropriate file permissions
5. **Signature format**: Base64 encoding for transport safety

### Security Boundaries

1. **No network operations**: Eliminates network-based attacks
2. **No key derivation from passwords**: Keys are randomly generated
3. **No automatic key distribution**: Manual public key sharing
4. **File system permissions**: Private keys readable only by owner
5. **Input validation**: All inputs validated before cryptographic operations

### Threat Model

**Protected against:**
- Content tampering (cryptographic signatures)
- False attribution (public key verification)
- Log tampering (hash chaining)

**NOT protected against:**
- Private key theft (file system security responsibility)
- Social engineering (human verification of agent identity)
- Quantum computing attacks (RSA limitation)

### Security Testing Requirements

1. **Signature forgery resistance**: Cannot create valid signatures without private key
2. **Hash collision resistance**: Content changes detectable via hash
3. **Key generation entropy**: Keys are cryptographically random
4. **Input fuzzing**: Malformed inputs handled safely
5. **File permission tests**: Private keys protected appropriately

## Approved Dependencies

### Python Standard Library Only

**Cryptographic modules:**
- `hashlib`: SHA-256 hashing
- `os`: Secure random number generation (`os.urandom`)
- `base64`: Encoding for signature transport

**Data handling:**
- `json`: Signature and log serialization
- `time`: Timestamp generation

**CLI and I/O:**
- `argparse`: Command-line interface
- `sys`: Exit codes and system interface
- `pathlib`: File path handling

**Cryptographic operations:**
- `cryptography` library: **EXPLICITLY APPROVED** for RSA operations
  - Rationale: Python stdlib has no RSA implementation
  - Scope: Only for key generation, signing, verification
  - Alternative considered: Implement RSA from scratch (rejected due to security risks)

### Dependency Policy

1. **Minimize external dependencies**: Use stdlib when possible
2. **Security-critical dependencies**: Only well-maintained, audited libraries
3. **Version pinning**: Lock dependency versions for reproducible builds
4. **License compatibility**: All dependencies must be BSD/MIT/Apache compatible
